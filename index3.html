Atributos, Tipos y Cardinalidades

Los Detalles que Definen los Datos

Atributos y sus Tipos

Los atributos son las características o propiedades que describen una entidad en una base de datos. Cada atributo tiene un tipo de dato asociado que define qué tipo de información puede almacenar.

Tipos de Datos Comunes

  • Numéricos: Enteros (INT), Decimales (FLOAT, DOUBLE), etc.
  • Texto: Cadenas de caracteres (VARCHAR, CHAR, TEXT).
  • Fecha y Hora: Almacenan fechas (DATE), horas (TIME) o ambas (DATETIME).
  • Booleanos: Representan valores verdadero/falso (TRUE/FALSE).
  • Otros: Binarios (BLOB), enumeraciones (ENUM), etc.

La elección del tipo de dato adecuado es crucial para garantizar la eficiencia y la integridad de los datos almacenados.

Cardinalidades en las Relaciones

Las cardinalidades describen cuántas instancias de una entidad se relacionan con cuántas instancias de otra entidad. Las cardinalidades más comunes son:

Uno a Uno (1:1)
Cada instancia de una entidad se relaciona con una y solo una instancia de otra entidad. Ejemplo: Una persona tiene un solo número de pasaporte.
Uno a Muchos (1:N)
Una instancia de una entidad se relaciona con muchas instancias de otra entidad. Ejemplo: Un cliente puede realizar muchos pedidos.
Muchos a Muchos (N:M)
Muchas instancias de una entidad se relacionan con muchas instancias de otra entidad. Ejemplo: Un estudiante puede estar inscrito en muchos cursos, y un curso puede tener muchos estudiantes.

Las cardinalidades son importantes para diseñar la estructura de la base de datos y garantizar que las relaciones entre las entidades se representen correctamente.
